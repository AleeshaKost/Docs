Jelaskan bagaimana seseorang mulai menggunakan produk Anda. Apa hal pertama yang perlu mereka lakukan? Apakah ada prasyarat?

Panduan *quickstart* sebaiknya membawa pengguna dari titik nol hingga bisa menggunakan produk Anda. Mereka akan mendapatkan keberhasilan awal yang cepat serta gambaran mengenai apa saja yang bisa mereka capai.

## Prasyarat

Sebelum memulai, Anda harus memiliki:

- Persyaratan pertama (misalnya, Node.js 18+, akun gratis, kunci API)
- Persyaratan kedua (misalnya, perangkat yang kompatibel, peramban yang kompatibel, sistem operasi yang kompatibel)

## Mulai

<Steps>
  <Step title="Install">
    Jelaskan cara menginstal produk Anda atau cara mendaftar.

    ```bash
    npm install package-anda
    ```
  </Step>
  <Step title="Konfigurasi">
    Jelaskan pengaturan atau konfigurasi yang diperlukan sebelum penggunaan pertama.

    ```bash
    cli-anda init
    ```
  </Step>
  <Step title="Jalankan">
    Tampilkan hal pertama yang dilakukan pengguna untuk melihatnya bekerja.

    ```bash
    cli-anda start
    ```
  </Step>
</Steps>

<Tip>
  Berikan cara bagi pengguna untuk mendapatkan bantuan. Ini bisa berupa tautan ke halaman dukungan, fitur obrolan dengan agen layanan pelanggan, atau forum untuk produk Anda.

  Contoh: Butuh bantuan? Hubungi kami di [support@yourcompany.com](mailto:support@yourcompany.com).
</Tip>
